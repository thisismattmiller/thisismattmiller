Giallo

Using a vision language model to analyze Italian Giallo films

Oct 31 2025

Intro

Italian Giallo Films seem to be having a little resurgence recently. I first was exposed to them when the Criterion Channel app had a collection of them for Halloween a few years ago and now watch a few of them around this time of year mixed in with other spooky season movies. They are terrible movies, but are usually visually striking. It’s like the cinematographer or director is competing to make the most saturated, dramatic and elaborate scene possible. Coupled with the 1960/70s aesthetic they are really kind of fun to visually experience. Watching them you notice recurring tropes, things like extreme zoom shots of the actors eyes, a shadow creeping up a staircase or a POV knife wielding gloved hand. I was wondering if you had a large corpus could you automatically group these tropes and maybe discover new ones or even see influences?

Image Similarity algorithms are a pretty common approach to work with a corpus to find visually similarity but I wanted to group the content of the scenes not just how they looked. I want to group two scenes together of a shadow walking down a hallway even if they are visually very different. So I thought I could maybe use a vision language model to describe a scene and then use that textual description to group and explore. I decided to try and use the Qwen2.5-VL-72B-Instruct model and send it a frame from the movie every 5 seconds to build a textual representation of the film. I did this for about 70 Giallo films and used those texts to do some experiments. This post looks at that process and results.

Code + Data

All the code and data that I can make public is in this repo. This project is working with copyright material though and also it's pretty graphic so I’m limited in what I can realistically put up in the form of public tools and visualizations. But everything I created from the material such as scene descriptions and embeddings I put in the repository.

This_Is_Cinema.mkv

In any project collecting the data is often the hardest part. If you want to algorithmically examine Giallo films you need a lot of Giallo films. Wikipedia has a helpful list of films I decided to just focus on the 1970s films which really represent the peak I think of the genre. I spent some time gathering the ~140 films from the list as best I could. I ended up only able to get 70-ish of them in good quality. I was pretty discerning, a lot of these are on free streaming sites but they are in potato quality. However even only 70 films if you extract a frame every 5 seconds ended up being over 80,000 images. I thought this was enough to get a sense if it would work at all. Process

I ran all 80k images through the model with a prompt asking it to identify things in the frame, striking features of the scene and a few sentence description. Here is an example of what a single frame data looks like: 

"The.Bird.With.The.Crystal.Plumage.1970.mkv__0465.jpg": {
    "tags": [
      "man",
      "glasses",
      "telephone",
      "hand",
      "face"
    ],
    "striking": [
      "closeup of eyes",
      "dark lighting",
      "intense gaze"
    ],
    "description": "Man holding telephone to ear. Eyes focused and serious. Dark shadows cast on face.",
    "file_name": "The.Bird.With.The.Crystal.Plumage.1970.mkv__0465.jpg"
  }

In this case the count in the filename 0465 is the timestamp of the film 0465*5 seconds == 38 Minute and 45 Seconds

At this point I built some tools that lets you explore the 70 film corpus textually by features. You can see across all film scenes that have "closeup of eyes" for example. Here is a video showing that interface if you are interested since I cannot put it up publicly (source code in repo though).

Embeddings

Next step was to send the 80K frames to build embeddings. I used the Google gemini-embedding-001 in “cluster” mode and only used the “tags” and “description” section of the text to build the embedding.

Now with embeddings available I first built nearest neighbors for all the frames and created a little interface to let me manually browse those, again I can't put it online so here is video if interested.

Expand to watch video

But I wanted to automatically find tropes, I’ve built clusters before after reducing the 3072 dimensions using something like UMAP but I wanted to try building clusters without reducing dimensionality. I tried HDBSCAN which is interesting because it doesn’t force every vector into a cluster which makes sense for this application I wouldn’t expect every frame to be related to another.

The result was around 330 clusters which I sent off the descriptions of each cluster to a LLM to organize them into a single description of what all the descriptions had in common. The result is a pretty interesting list of possible scenes you would expect to find in a Giallo:

Hallucinations

Of course if we are dealing with vision language models there are going to be problems. I noticed that sometimes I would rarely get a frame with a completely wrong description. I found that the VLM would generate a predictable description for some frames especially if it was an abstract or completely black image. It would just invent a realistic sounding description like “a masked figure lurks in the shadows. the woman's eyes widen in terror. blood stains the pristine white dress.” It would reuse the same duplicate wrong descriptions for many images so I could aggregate on them and find them. If you want examples you can see in this tool (loads 45MB of data) [https://thisismattmiller.github.io/giallo/apps/dupe-descriptions/]. The takeaway is that if you are using these models for “serious” work you better have a seriously good QA workflow process because it will generate bad data eventually.

Plotting the Tropes

I wanted to see if you could plot these clusters within the playtime of the movies. I was thinking maybe you could see that specific types of scenes happen a predictable times in these types of movies. For example something like, you often see the silhouetted killer creeping up the staircase at the half-way mark of the movie. I built a tool to plot the occurrences of the cluster members on a 0-100% x-axis. Technically it works, for example here is the plot for the cluster: “Cluster 3 - Opening credit sequences featuring airplanes and production details.”

If you explore there are interesting patterns you can extrapolate, for example: “Cluster 120 - Close-up depictions of men with bloodied faces exhibiting intense pain and distress.”

Which makes sense in the context of these movies often the male antagonist gets their comeuppance or justice for their crimes in the last act of the film.
